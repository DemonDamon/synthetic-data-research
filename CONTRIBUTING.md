# 贡献指南 (Contributing Guide)

感谢您对 **Synthetic Data Research** 项目的关注! 我们欢迎并感激任何形式的贡献,无论是内容补充、错误修正还是新想法的提出。

## 如何贡献

### 1. 报告问题 (Issues)

如果您发现了文档中的错误、不准确之处或有改进建议,请通过 [GitHub Issues](https://github.com/DemonDamon/synthetic-data-research/issues) 提交。

**提交Issue时请包含**:
- 清晰的问题描述
- 问题所在的文件路径
- 如果是错误,请提供正确的信息或参考资料
- 如果是建议,请说明改进的理由

### 2. 提交Pull Request

我们欢迎您直接提交代码或文档的改进:

**步骤**:
1. Fork本仓库到您的GitHub账号
2. 创建一个新的分支 (`git checkout -b feature/your-feature-name`)
3. 进行您的修改
4. 提交更改 (`git commit -m 'Add some feature'`)
5. 推送到您的分支 (`git push origin feature/your-feature-name`)
6. 创建Pull Request

**Pull Request要求**:
- 清晰描述您的更改内容和目的
- 如果是内容补充,请确保引用了可靠的来源
- 遵循现有的文档格式和结构
- 检查拼写和语法错误

### 3. 内容贡献

#### 添加新的研究报告

如果您想添加新的研究主题或深入分析:

1. 确定合适的目录位置 (参考现有结构)
2. 创建必要的子目录和`images`文件夹
3. 撰写`README.md`和`detailed_report.md`
4. 确保包含参考文献和引用来源
5. 在上级目录的README中添加链接

#### 补充案例研究

实践案例非常有价值,如果您有实际应用经验:

1. 在相应的应用领域目录下创建`case_studies.md`
2. 描述问题背景、解决方案、结果和经验教训
3. 如果可能,提供代码链接或示例
4. 说明该案例的适用场景和局限性

#### 推荐资源

如果您发现了优质的论文、工具或教程:

1. 在`09-academic-resources`相应子目录中添加
2. 提供简短的描述和为什么推荐
3. 确保链接有效且资源可访问

## 内容规范

### 文档格式

- 使用Markdown格式
- 遵循现有的标题层级结构
- 使用代码块标注公式和代码
- 适当使用表格组织信息
- 图片放在`images`文件夹并使用相对路径引用

### 引用规范

- 所有事实性陈述必须引用来源
- 使用Markdown链接格式: `[描述](URL)`
- 在文档末尾列出所有参考文献
- 优先引用学术论文、官方文档和权威资源

### 语言风格

- 使用清晰、专业的语言
- 避免过度使用行话,必要时提供解释
- 中英文混合时注意格式规范
- 保持客观中立的叙述风格

## 审核流程

1. **初步审核**: 维护者会检查您的贡献是否符合基本要求
2. **内容审核**: 评估内容的准确性、完整性和价值
3. **格式审核**: 确保符合文档规范和项目结构
4. **合并**: 审核通过后合并到主分支

## 行为准则

- 尊重所有贡献者和用户
- 保持建设性和友好的讨论氛围
- 接受建设性的批评和建议
- 遵守学术诚信,不抄袭或剽窃他人成果

- 请确保您有权分享所提交的内容
- 引用他人工作时请注明出处
